# Fanoir 이미지 업로드 가이드

## 권장 사양

| 용도 | 비율 | 권장 해상도 | 최대 업로드 | 최적화 후 목표 |
|------|------|-------------|-------------|--------------|
| 상품 이미지 | 1:1 (정방형) | 1000 x 1000 px | 5 MB | ~200 KB |
| 컬렉션 배너 | 21:9 (가로형) | 2400 x 1029 px | 10 MB | ~400 KB |

## 자동 최적화 프로세스

업로드 시 서버에서 자동으로 다음 처리가 적용됩니다:

1. **리사이즈**: 권장 해상도에 맞게 `cover` 모드로 리사이즈 (비율 유지 + 중앙 크롭)
2. **WebP 변환**: 모든 이미지를 WebP 포맷으로 변환 (60~80% 용량 절감)
3. **S3 업로드**: 최적화된 파일을 S3에 저장

> `withoutEnlargement: true` — 원본이 권장 해상도보다 작으면 억지로 키우지 않습니다.

## 용량 참고표

| 구분 | 원본 JPEG | 원본 PNG | 최적화 후 WebP |
|------|----------|---------|--------------|
| 상품 (1000x1000) | 200~500 KB | 500KB~2 MB | **~150~250 KB** |
| 배너 (2400x1029) | 500KB~2 MB | 2~5 MB | **~300~500 KB** |

> 클라이언트에서는 `next/image`가 디바이스에 맞게 추가 리사이즈를 자동 수행합니다.

## 허용 파일 형식

- **JPEG/JPG** — 사진에 가장 적합 (권장)
- **PNG** — 투명 배경 필요 시에만
- **WebP** — 이미 최적화된 경우

## 사용처별 적용

| 컴포넌트 | 이미지 타입 | `sizes` 속성 | 비고 |
|----------|-----------|-------------|------|
| Showcase 그리드 | 상품 (1:1) | `(max-width: 768px) 50vw, 33vw` | 6개 정방형 그리드 |
| Showcase Signature | 상품 (1:1) | `(max-width: 1024px) 50vw, 25vw` | 상위 2개 상품 재사용 |
| Collection 상품 카드 | 상품 (1:1) | `(max-width: 768px) 50vw, ... 25vw` | 2~4열 반응형 |
| CollectionBannerSlider | 배너 (21:9) | `(max-width: 768px) 100vw, 50vw` | 슬라이더 |
| CollectionDetail Hero | 배너 (21:9) | `100vw` | `priority` 적용 |

## 촬영/준비 팁

- **상품**: 정방형 프레임으로 촬영. 직사각형이면 중앙 기준 크롭됨
- **배너**: 21:9 가로형 구도. 하단에 텍스트 오버레이가 들어가므로 **핵심 피사체는 중앙~상단에 배치**
- **PNG < JPEG**: 사진은 JPEG가 원본 용량이 작아 업로드 속도 우수. 투명 필요한 경우만 PNG
- **과도한 고해상도 지양**: 4000px 이상은 업로드 시간만 늘어남. 권장 해상도의 1~2배면 충분

## S3 저장 경로

```
https://fanoir.s3.ap-northeast-2.amazonaws.com/images/{timestamp}-{filename}.webp
```

어드민 > Image Guide 페이지에서 동일한 내용을 시각적으로 확인할 수 있습니다.
